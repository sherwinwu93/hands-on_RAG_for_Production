# Chapter 1. Introduction to Retrieval-Augmented Generation (RAG)
RAG: 原来LLM只能回答提前训练的知识,改成能回答私密~特定领域~最新的知识

## How does RAG work?
RAG steps:
1. R(retrieval): search and retrieve relevant documents
2. G(generation): LLM receives original query and retrieved documents to generate a response
    A(Augmented)指的是把retrieved document结合original query给LLM
```python
# RAG basic prompt
"""
You are an assistant for question-answering tasks. Use the following
pieces of
retrieved context to answer the question. If you don't know the
answer, just say
that you don't know.
<question>
{question}
</question>
<context>
{context}
</context>
Answer:
"""
```

直接问LLM相当于闭卷考试,RAG相当于开卷考试(可以查private,domain-specific,latest data)

## The Blueprint of a RAG Stack
RAG flow:
1. Indexing: 相当于存
    各种文档 --Metadata-> Indexing API ---> Extract text Chunking Table processing ---> Embedding ---> Vector Store
2. Query: 相当于查
     Query --params Filters-> Query API --Embedding-> Vector Store --Hybrid Search ---> Reranking ---> Prompt --LLM-> Hallucination detection ---> Response

### The Ingestion Flow
indexing or embedding: 把文本转成向量存起来(但不单单向量)
生产indexing遇到的问题:
- 文档预处理
- 文档分块
- 文档向量化
- 数据校验
- 多模态输入的处理(声音,图片,视频)
- 增量的更新

### The Query Flow
搜索技术:
- 语义搜索(semantic search)
- 词汇搜索(lexical search)

搜索flow:
1. Query得到相关文档(Retrieval)
2. 把文档和问题通过prompt,输入给LLM,LLM输出(Generation)
    还有回答的引用自哪里的文档
3. 校验LLM的回答,是否根据相关文档生成,是否有害(Guardrails)

生产问题:
1. 如何不单单只向量搜索,用Hybrid search, reranking, 提升搜索效果?
2. 如何把表格,图片,图表,其他多模态数据喂给管道,且保持准确性?
3. 如何测量RAG的质量, 不单单第一次发布,而且在更新后?
4. 如何集成agentic workflow和graph
--------------
生产的运维问题: 低延迟,高可用,安全
1. CI/CD for RAG:
   1. Automated data refresh 自动数据更新: 在知识库更新新文档
   2. Automated evaluation: 自动评估性能
   3. Prompt and model controls: prompt的版本控制(出问题可以迅速降级回滚)
   4. Observability and monitoring: 可以debug request, 监控各模块的cost
2. 性能和可伸缩性
   1. Database optimization
   2. Optimized inference: 用更恰当的模型做推理
3. 安全和管控
   1. Data-centric access control: RBACs(role-based access controls)
   2. Encryption at rest and in motion: 访问REST时加密
   3. Prompt and output sanitization: 防注入,防加密信息泄露
